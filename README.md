2. 브랜치 생성
    1. feature issue에 대한 branch
        1. feature/#이슈번호-기능
        2. 예) feature/#1-login
    2. refactor issue에 대한 branch 
        1. refactor/#이슈번호-기능
        2. 예) refactor/#107-**unify-product-name-field**
3. 개발 완료 후, **main 브랜치**로 pull request
4. 코드 리뷰
5. **MERGE**
